Hooks provide a more direct API to the React concepts you already know:  
props, state, context, refs, and lifecycle. 

---
# Motivation

---
# It’s hard to reuse stateful logic between components

React doesn’t offer a way to “attach” reusable behavior to a component. 
Hooks allow you to reuse stateful logic without changing your component hierarchy

---
# Rules
<ul>
  <Appear>
    <ListItem>Only Call Hooks at the Top Level</ListItem>
    <ListItem>Only Call Hooks from React Functions (or Other Hooks)</ListItem>
  </Appear>
</ul>

---
# Stateful
<ul>
  <Appear>
    <ListItem>Two components using the same Hook <i>do not</i> share state.</ListItem>
    <ListItem>All state and effects inside of it are fully isolated.</ListItem>
  </Appear>
</ul>
